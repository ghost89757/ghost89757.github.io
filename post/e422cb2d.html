1 常用UI控件

  • 控件：界面系统的术语，指一个具有独立状态、外观和操作的对象，如按钮、输入框、文本标签、图片等

  • 在Unity中，控件就是一个游戏物体，在UI界面上，并且由挂载了不同组件的不同父子物体组成。

  • 一种控件，往往挂载一种同名核心组件，比如按钮控件，他是一个控件，包含了多个子物体，自身也挂载了多个组件，自身的组件中有一个用于实现按钮功能的核心组件，叫做按钮组件：Button。如下图所示，区别控件Button和组件Button。

1.1 创建游戏界面

  • 创建一个Button控件：右键 -> UI -> Button

    • 自动生成：画布(Canvas)、按钮控件(Button)、事件系统(EventSystem)

  • 画布、控件、事件系统

    • 所有UI控件都是Canvas的子物体，方便统一布局
    • 允许存在多个Canvas，形成多个独立界面层
    • UI系统中的交互会用到事件系统

  • 界面比例

    • 在Game窗口中可以看到铺满全屏的UI，但在Scene窗口中UI以巨大的比例展现
    • UI系统以Canvas为载体，Canvas与显示设备的屏幕有直接对应关系。UI以1像素为单位，三维场景以1米为单位，若显示屏幕是1920像素1080像素，则对应三维场景大小就有1920米1080米
    • 由于UI在Scene窗口里的巨大界面，Unity在Scene窗口上提供了2D按钮，以便于对UI进行操作

  • 搭建测试场景

    • 自定义导入图片：（1）将图片导入Project窗口或Assets文件夹 （2）在图片的Inspector窗口中将Texture Type设置为Sprite(2D and UI)

1.2 矩形变换组件：Rect Transform

  • Rect Transform组件是Transform组件的子类，Rect是Rectangle的简写
  • 用Rect Transform而不用Transform，是因为页面控件的位置、大小比其他物体更复杂
  • 用多种相对参数取代了绝对的位置参数
    • 对齐情况
      • 参数：控件中心与父控件对齐中心的偏移量
    • 拉伸情况
      • 参数：控件边缘与父控件边缘的偏移量

1.5 按钮（Button）

Button控件

  • 用途：与玩家进行点击交互
  • 物体组成：

Button组件

  • 组件属性
    • Interactable：是否可交互
    • Transition：外观状态切换，每个按钮四种状态：普通、高亮、按下、禁用，不同状态的外观应有区别，定义外观变化有几种方式
    • Navigation：导航顺序，切换UI控件的顺序，大部分情况用不到
    • On Click()：点击按钮事件
  • 按钮外观状态切换方法
    • Color Tint：颜色叠加，不同按钮状态叠加不同颜色
      • Normal Color：默认状态的颜色。
      • Highlighted Color：鼠标悬停或按钮被选中时的颜色。
      • Pressed Color：按钮被按下时的颜色。
      • Selected Color：按钮被选中时的颜色（注意：Unity 的 Button 组件本身没有直接的 Selected Color，但可以通过代码实现）。
      • Disabled Color：按钮被禁用时的颜色
    • Sprite Swap：切换精灵图，不同按钮状态使用不同图片
  • 点击事件：OnClick，事件关联可在编辑器中直接操作
    • 四个选项从左到右，每行依次代表：（1）事件可以运行的事件段（游戏中、编辑器、两者都）（2）事件发生时调用的函数（3）事件操作的物体（4）事件的标签参数
    • OnClick事件可以触发任意物体的任意方法

1.6 单选框（Toggle）

Toggle控件

  • 用途：与玩家进行勾选交互
  • 物体组成：

Toggle组件

  • 组件属性：

    • Interactable、Transition、Navigation，三者使用与Button组件类似
    • Is On：是否处于勾选状态
    • Toggle Transition：勾选时是否有“淡入淡出”的效果
    • Graphic：对勾图片对应的子物体，默认CheckMark子物体，可修改
    • Group：代表本勾选框所属的勾选框组，一组有多个勾选框
    • On Value Changed(Boolean)：勾选事件，与Button触发点击事件类似

  • 单选框组Group的使用

    • 一个画布中，创建一个空物体Group1，并添加Toggle Group组件
    • 空物体中，创建若干个Toggle控件，每个控件的Toggle组件->Group设置为Group1
    • 至此，若干个Toggle控件属于一个单选框组Group1

1.7 滑动条（Slider）

Slider控件

  • 用途：显示和编辑一定范围内的数据，如游戏中的血条

  • 物体组成：

    • Background：滑动条的整体外观

    • Fill Area：已填充区域，滑动圆圈已滑过的区域

    • Handle Slider Area：手柄滑动区域，经实践应是滑动圆圈区域

Slider组件

  • 组件属性：

    • Interactable

    • Transition

    • Navigation

    • Fill Rect：已填充区域，调用子物体中的Fill Area

    • Handle Rect：手柄滑动区域，调用子物体中的Handle Slider Area

    • Direction：滑动方向，有四种：（1）从左到右（2）从右到左（3）从顶到底（4）从底到顶

    • MinValue：滑动条能取的最大值

    • MaxValue：滑动条能取的最小值

    • Whole Numbers：滑动条取值只能为整数

    • Value：滑动条所代表的值，根据滑动情况可以代表不同的值

    • On Value Changed：触发事件

2 事件系统

  • 创建UI画布后，会自动创建另一物体：事件系统（EventSystem），界面上各个控件的响应事件，都离不开EventSystem的支持。
  • EventSystem并非UI专用的，而是一个基础系统，为其他系统提供支持。
  • EventSystem提供了向游戏发送消息的途径，这些消息通常是输入消息，包括键盘、鼠标、触摸和自定义输入事件；EventSystem是事件的管理者，并非具体处理者，其包含一些列组件，互相配合，以达到管理和触发事件的功能。
  • EventSystem的功能包括
    1. Message System（消息分发系统）
    2. Input Modules（输入模块）
    3. 提供多种常用输入事件接口
    4. 管理各种射线

  2. PointerEventData：表示指针滑动、单击的数据

2.4 事件触发器（Event Trigger）

除了对物体添加脚本编写实现输入事件的接口，还可以使用Event Trigger组件，添加该组件后，单击Add New Event Type按钮，就可以添加任意事件了。

添加事件类型以后，响应函数的指定方法和按钮、输入框等控件是类似的。只是响应函数多了一个BaseEventData类型的参数，可以用它获取相关的事件信息，而很多时候需要注意的都是鼠标和轴相关的信息，可以通过类型转换的方法把它们转换为派生类变量，进而获取到更多有用的信息。
