3D RPG 游戏总结

发表于2025-04-12|更新于2025-04-12|游戏开发Unity
|字数总计:20.6k|阅读时长:83分钟|阅读量:

大概一个月前做的一个3D RPG游戏，以下是记录和总结，当然这中间碰到了许多问题，但好在每次遇到问题耐下心来总是能够解决的，解决完感觉世界的景色都美好了，现在回看，挺有成就感的。

1 创建项目并导入素材

  • 将现有3D项目升级URP

    1. 在包管理中引入Universal RP，在Project中创建URP资源

    2. 在ProjectSettings中，Graphics和Quality设置新建的URP资源

  • 导入素材，场景素材使用Low-Poly Simple Nature Pack，主角素材使用Dog Knight PBR Polyart

  • 素材适配当前渲染管线，所有素材升级到URP

    • 将导入的素材中的URP双击并导入

    • 版本变化：把材质付给建模，2022年已经不一样。 Window->Rendering->Render Pipeline Converter，打开窗口后下拉选择Build in，里勾选前两个，再initialize Converters->Convert Assets

2 尝试熟悉基本工具

  • 修改天空盒子

    • 导入天空盒子资源

    • 选中Window->Rendering->Lighting->Environment->Skybox Material

    • 用导入的资源替换

  • 拖入草地，选中配置的URP资源进行一些阴影设置

  • 渲染光照

    • Window->Render->Lighting->Scene->NewLightingSettings，创建一个新的光照设置
    • Lighting->Scene->MixedLighting->LightingMode可选光照模式
    • Lighting->Scene->WorkflowSettings->GPUBakingDevice可选运行的显卡
    • 右下角GenerateLighting进行光照渲染
    • Lighting->Environment->EnvironmentLighting可调试出不同的环境光

  • 在场景中摆放环境预制体

    • 技巧1：V键+鼠标左键，可以轻松是预制体适配地形
    • 技巧2：ctl+shift+f，可以使得选中物体移动到当前场景屏幕视角
    • 技巧3：使用空物体当作文件夹，管理场景中的物体
    • 技巧4：crl+shift+鼠标左键，物体可以以吸附地形的形式自由移动

3 PolyBrush 发挥创意构建场景

PolyBrush

  • 使用PolyBrush，可以修饰地图，如地形、软化、上色、快速点缀环境预制体等

  • 在包管理窗口导入PolyBrush，Sample选择URP

  • 在Project中，创建Addons文件夹，用于存储工具包文件

  • 打开PolyBrush，菜单栏 -> Tools -> PolyBrush -> PolyBrushWindow

  • 尝试PolyBrush的功能

    • 功能3（刷颜色）：屏幕中地图网格的材质和我们颜色刷的材质不匹配，在Project中选中Polybrush -> Shader Example -> Shader -> LitVertexColorURP，创建Material并添加到地图上，此时材质匹配，可以用Polybrush在地图网格上刷颜色
    • 功能4（刷预制体）：
      • 在CurrentPalette中添加要刷的预制体
      • 注意选项Hit Surface is Parent，这表示刷上的预制体是被刷物体的子物体
      • 注意选项Use Pivot，不选的话刷上的物体不吸附地面而漂在空中

ProBuilder

ProBuilder

  • 删除原来的地图，使用ProBuilder插件来搭建，可自定义自己想要的地图，包括地图大小、坐标、顶点数量

  • 在包管理窗口导入ProBuilder，Sample选择URPSurpport；放入Addons文件夹以便于管理；在菜单栏Tools选项可以找到ProBuilder Window，可选择Window右上角Icon Model便于显示功能；

  • 使用ProBuilder可以创建具备指定顶点、形状、大小的空间物体

  • 可以选择三角形图案功能，将网格形状有四边形变为三角形

  • 给新建地图添加PolyBrush匹配的材质：新建一个材质，材质的Shader选项选择Polybrush -> Shader Example -> Shader -> LitVertexColorURP

ProGrids（可选）

  • 为了配合使用，在包管理窗口中导入ProGrids，2021版之后，在包管理窗口中，左上角加号Add package from git URL并输入com.unity.progrids，以此导入该包；使用ProGrids可以以指定的网格单位移动物体

4 Navigation 智能导航地图烘焙

  • 打开Navigation：菜单栏 -> Window -> AI -> Navigation

烘焙地面

  • 设置地面为Navigation Static
    • 方法1：选中Ground，Inspector窗口中，Static选项选择Navigation Static，不应用到子物体
    • 方法2：选中Ground，Navigation窗口中，Object -> Ground -> Navigation Static，不应用到子物体
  • 设置地面为Walkable：选中Ground，Navigation窗口中，Object -> Ground -> Area，选择Walkable
  • 烘焙：选中Ground，Navigation窗口中，Bake -> 右下角，点击Bake开始烘焙

烘焙树木

  • 选择场景中所有的树木，技巧：在Hierarchy窗口中搜索“Tree”开头的物体，点击第一个，shift并点击最后一个
  • 设置为Navigation Static，Not Walkable，并且烘焙
  • 使用2021版本遇到问题：全选树木后只能选Navigation，不能选择Area是否Walkable，解决方法：Navigation -> Object -> Mesh Renderers，全选与Tree有关的，即可选择Area，然后进行烘焙

烘焙主角

  • 主角增加组件：Nav Mesh Agent，高设置为1.4、半径设置为0.35
  • 烘焙：将烘焙的高、半径与Agent保持一直，进行烘焙，烘焙后树木不可走区域可以看到变小

烘焙障碍物

  • 障碍物增加组件：Nav Mesh Obstacle，可勾选Crave，将为该障碍物实时切割出不可走区域

9 ShaderGraph 遮挡剔除

ShaderGrapsh创建

  • 要实现人物在树木或其他遮挡物遮挡时可以显示出轮廓

  • 创建一个ShaderGraph：在资源文件夹Material中创建子文件夹ShaderGraph，并在子文件夹中右键 -> Create -> ShaderGraph -> URP -> Unlit Shader Graph，并修改其名称为Occlusion Shader，并为其创建一个材质放入Material文件夹下

  • Fragment增加Alpha通道：小窗口Graph Inspector -> Universal -> Alpha Clipping，勾选选项

  • 增加Fresnel Effect节点：右键 -> Create Node -> 搜索名字，是自带的光圈效果，使人物被遮挡后可以看到轮廓

  • 增加可调节的颜色：小窗口Occlusion Shader -> +号 -> Color类型的变量，取名就叫Color；点击该变量Color，可在小窗口Graph Inspector -> Node Settings中调整相关属性，比如Default可以自定义颜色；从Occlusion Shader窗口将Color拖拽到编辑区域中，可以得到一个Color节点

  • Fresnel Effect节点与Color节点做乘法效果：新增一个Multiply节点，Fresnel Effect节点输出连Multiply输入A，Color节点输出连Multiply输入B，Multiply输出连Fragment节点Base Color，此时可以看到光圈和节点的乘法效果

  • 增加Dither节点：新增一个Dither节点，为其创建一个float类型的变量取名为DitherCount，输出连Fragment的Alpha，使得光圈的实心填充效果可以调整

  • 新建一个float类型的变量AlphaThreshold，用于连接控制Fragment的Alpha Clip Threshold

  • 点击左上角Save Asset，此时为该ShaderGraph创建的材质也具备了相应的效果，并且可以进行相关属性的调整

将ShaderGraph的材质应用到人物

  • 在Assets窗口中，选中PipelineSettings -> New URP Asset_Renderer；在Inspector窗口中，Add Renderer Freature -> Render Objects并取名CharacterBehind，在Overrides中Material使用ShaderGraph创造的材质、Depth勾选 -> Write Depth取勾，Depth Test选Greater -> 根据深度判断人物被树木遮挡；为人物增加图层Player，并在Filters -> Layer Mask中选择Player图层
  • 在Assets窗口中，选中PipelineSettings -> New URP Asset_Renderer，在Inspector窗口再增加一个RendererObjects，并将图层选为Player，此时人物不被遮挡时将不会出现轮廓

问题：单例管理器重复

Player从场景A进入场景B，再场景B回到场景A时，DontDestroyOnLoad中出现重复的GameManager、MouseManager、SceneController并且三个没脚本三个有脚本且GameManager脚本参数为空，点击场景中的地面Player无法移动

根本原因： 泛型单例实现错误

在泛型单例脚本Singleton中，有一个致命的逻辑错误，错误如下

根据错误的代码，当单例不同时，我们只是销毁了instance指向，即销毁了脚本而并没有销毁物体。

错误泛型单例下跨场景发生了什么

运行场景A
  1. 保存管理器：场景A预设的单例管理器物体GameManager、MouseManager、SceneController会运行自身的脚本，由于执行Awake()的DontDestroyOnLoad()它们将被标记为DDOL，在转换场景时不会销毁
  2. 生成玩家：我们在运行前取消勾选PlayerA运行后再勾选，模拟了玩家的生成
  3. 管理器数据引用：玩家将会进行进行事件订阅和管理器注册，此时GameManager将引用PlayerA的数据
场景A转换到场景B

  1. 异步加载场景：异步加载场景会首先销毁场景A中的对象（DDOL的除外），然后加载场景B中预设的对象；由于我们在场景B中并未预设单例管理器，所以从场景A到场景B单例管理器无须进行单例唯一性更新，这里所说的单例唯一性请参考以下代码

  2. 生成玩家：场景B中也无预设玩家，根据SceneController中的跨场景逻辑，会根据预制体生成新的PlayerB

  3. 管理器数据引用：并进行实践订阅和管理器注册，此时GameManager将引用PlayerB的数据

场景B转换到场景A
  1. 异步加载场景：销毁场景B中的对象（DDOL的除外），加载场景A中的预设对象；
    • 注意，场景A中的预设对象包括三个单例管理器物体，我们姑且称新加载的预设对象为ManageNew，它们首先会执行自身的脚本中的Awake()并产生两种效果：1）首先执行父类的Awake()，判断单例唯一性发现单例不为空，将执行Destroy(instance)，于是销毁了自身的单例脚本，成为了没有脚本的物体，即僵尸管理器 2）执行子类自身的DontDestroyOnLoad(this)，将自身标记为DDOL
    • 此时场景B保留的管理器Manage和场景A中新加载的僵尸管理器ManageNew便形成了重复
  2. 生成玩家：其实场景A中有了PlayerA，只不过我们运行场景A之前取消勾选了，所以加载时Player也被取消勾选了；根据跨场景的实现逻辑，我们根据预制体生成一个PlayerANew
  3. 管理器数据引用：
    • 对于场景B保留的管理器Manager，其引用的数据来自场景B中的PlayerB，并且由于场景转换PlayerB已销毁，此时Manager引用的玩家数据为空
    • 对于场景A新加载的管理器ManagerNew，由于泛型单例的错误实现，导致它们成为了没有脚本只有物体的僵尸管理器，其无法进行数据的引用
    • 因此，场景A中新生成的PlayerANew无法进行事件订阅和管理器注册，PlayerANew将无法进行移动

至此，从根本原因到实际问题的产生已经梳理明白
