1 动画系统基本概念

现代游戏动画系统的概念，简单介绍一下：

  • 帧：计算机每次渲染完一幅图并并显示出来，这一幅图就是一帧。
    • 帧率：连续的帧形成了动画，每秒刷新帧的次数称为帧率，单位是“FPS（Frames Per Second，帧/秒）”，也可以简称为“帧“
    • 帧间隔实践Time.deltaTime，当前帧与上一帧的实践差值
  • 帧动画：一种动画技术，事先准备好每一帧的画面，然后依次播放。简单直接，但工作量大，存储空间也大。
  • 2D/3D骨骼动画：人物模型会绑定一个骨架，每一段骨骼关联人物模型的一部分，移动和旋转骨架，人物也会跟着做各种动作。节省工作量和存储空间。
  • 动画融合：通过插值等技术，自动计算不同动作状态之间的过渡状态，使得状态转换的自然。
  • 动画状态机：表示了各种动画状态间的联系以及转换逻辑。

2 2D动画实例分析

2.1 素材准备

现在的商场里找不到Standard Assets了，苦了我这个菜鸟小白呀>^<，找了好久才发现已经下架了，好在还可以下载，放个链接：https://download.unity3d.com/download_unity/a6cc294b73ee/WindowsStandardAssetsInstaller/UnityStandardAssetsSetup.exe

2.1.1 资源导入与整理

导入资源

在窗口Project -> Assets文件夹下，导入本地已经下载好的资源包Standard Assets -> 2D.unitypackage

整理资源

  • Project -> Assets文件夹下，创建Resources文件夹
  • Resources文件下，创建anims和sprites文件夹，分别存放动画资源和图片资源
  • Assets/Standard Assets/2D/sprites文件下，PlatformWhiteSprite.png文件和所有Robot开头的png图片文件，复制Assets/Resources/sprites文件夹

2.1.2 图片设置

  • 参数设置：
    • Texture Type（贴图类型）：Sprite（2D and UI），即2D精灵图片
    • Sprite Mode（精灵模式）：（1）Single，一张图片文件里只有一张图（2）Multiple，一张图片文件里有多张图
    • 这里除了PlatformWhiteSprite.png是单图，其他图片文件均为多图
    • Pixels Per Unit（像素与图片大小的比例）：默认100，即100像素对应场景中的1米
    • 单击Apply，是设置生效
  • 批量选中图片：按住Ctrl/Shift键
  • 多图模式切割
    • 编辑器菜单栏Window -> Package Manager -> 左上角加号后边选择Packages: Unity Registry，找到2D Sprite并安装
    • 选中图片，点击Sprite Editor，打开精灵编辑器
    • 常用功能Slice（切割），3种类型：Auto（自动）、Grid By Cell Size（根据图片大小划分网格）和Grid By Cell Count（根据行数列数划分网格）
    • 这里我们选用第二种类型，切割大小设置为275像素*275像素
    • 点击Apply，使设置生效

2.2 场景搭建与角色控制

2.2.1 场景搭建

  • 新建场景：编辑器菜单栏File -> New Scene，保存并命名为Test2DAnim：File -> Save As，单击场景窗口2D按钮，切换到2D视角
  • 地板搭建：使用PlatformWhiteSprite图片，添加Box Collider 2D组件，并适当调整，做成预制体存放于Resources/prefabs文件夹下
  • 玩家Player：使用RobotBoyIdleSprite切割好的任意图片
    • 找到切割好的图片：Resources/sprites/RobotBoyidelSprite，点开类似于播放的小三角按钮
    • 添加Rigidbody2D组件，勾选Constraints选项中的Freeze Rotation Z选项，避免玩家跌倒
    • 添加Capsule Collider 2D组件，点击Edit Collider调整胶囊形状，胶囊体要比人物略小、左右对称、脚部对准

2.2.2 角色控制

对玩家Player进行操作：

  • 添加资源自带的通用控制角色运动脚本，Platformer Character 2D
  • 添加资源自带的输入脚本（键盘、手柄等输入），Platformer 2D User Control
  • 指定角色脚本的应用
    • 添加Animator组件，并在Resources/anims文件下新建一个Animator Controller文件，放入Animator组件的Controller一栏
    • 创建一个名为GroundCheck的子物体，放于玩家脚下，用于地面检测
    • 在角色脚本Platformer Character 2D中，What Is Ground选项，默认是Nothing（没有），将它改为Everything

至此，玩家可以进行左右移动。

2.3 动画的制作步骤

为角色加上动画，总的步骤分为3步：

  • 编辑动画：把素材中的图片串联起来，形成动画
  • 编辑动画状态机：把编辑好的动画，通过Animator有机结合起来，可以转换状态
  • 设置动画变量

2.3.1编辑动画

  • 打开Animation窗口：编辑器菜单栏Window→Animation→Animation
  • 创建动画文件：选中Player，在Animation窗口点击Create，自定义路径和文件名，如Resources/anims/Idel.anim，做站立动画。注意，Animation窗口播放的是当前选中物体的动画。
  • 放入动画素材：将Project窗口中，找到文件夹Assets/Resources/sprites/RobotBoyIdelSprite，将该文件夹下的所有分割好的图片拖拽到Animation拖拽到Animation窗口中，即可自动形成动画序列
  • 更改动画播放速率：Animation窗口的Samples（采样率、速率）参数，默认60，可适当更改
  • 播放动画：选中Player，点击播放按钮，即可播放；点击左右短箭头，也可以在场景中预览

由以上步骤，可以得到玩家的walk、run、jump等动画动作，到了这里我挺激动的，太兴奋辣~

2.3.2 编辑动画状态机

  • 打开Animator窗口：编辑器菜单栏Window→Animation→Animator
  • 增删状态：选中Player，Animator窗口中已有几个自动添加的新状态，通过右键状态或这拖拽动画文件，可以进行状态的增删。
  • 状态连接：通过右键一个状态选择Make Transition即可创建一个到下一状态的箭头。
  • 动画状态机中的状态：Entry是初始状态，是入口，其连接的第一个状态时默认状态;Any State时任意状态

2.3.3 设置动画变量

有了状态转移图（动画状态机），我们该考虑状态之间的转换逻辑，即箭头的问题。

  • 增加动画变量：在Animator窗口左上角，Paraments->在点小加号，即可增添不同类型的变量以及设置变量值

  • 设置过度逻辑：单击箭头，即可查看一状态到另一状态的转移信息。如下如所示，我们在增设了动画变量后，可以在Conditions中看到变量的有关信息，通过速度是否高于或低于0.2，0.01，我们可以实现站立、走、跑三种状态之间的转移逻辑；通过是否碰到地面，可以实现跳跃、站立、任意状态三者之前的转移逻辑。

至此，我们的角色在移动时有了动画动作。但是，实际运行时发现，延迟很高，这就涉及到了动画过渡。

2.3.4 动画过渡

如图，是一个动画状态转移信息，关于时间的参数有两部分：

  • Exit Time（退出时间），指切换到其他动画前，基于播放一段当前动画所用的时间。常用于不可中断的动画，如攻击收招、拔枪等动作。
  • Transition Duration（状态转换时间），指从状态A切换到状态B的叠加状态时间，从上图可直观看出。用于动作的完美过渡，如真实的3D动作游戏，玩家的动作切换。

但是我们现在的2D动画这两种设置所带来的效果，取消勾选图中的Has Exit Time，让这两部分置为0即可。

至此，我们可以得到一个无延迟的动画转换。

不足的是，我们使用导入的资源，完全写好了脚本，无法自己动手实现一个，需要自己再找个项目练一练手，以便体验脚本在动画方面的使用。
