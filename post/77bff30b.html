1 物理系统基本概念

物理引擎是一套基于牛顿力学的模拟计算系统，是对现实物理的一种近似模拟。

1.1 刚体

  • 在物理学中，在任何力的作用下，体积和形状都不发生改变的物体叫作“刚体（Rigid Body）”，刚体是力学中的一个抽象概念、理想模型

  • 有两种主要组件：Rigidbody组件和Rigidbody2D组件

  • 让物体产生物理行为的主要组件，一旦挂载，物体将受物理引擎控制，如受力等物理行为

  • 挂载刚体后，不易用脚本修改物体位置和朝向，可以通过力和速度的变化来影响物体的行为

  • 勾选刚体的运动学选项（Is Kinematic），物体将脱离物理系统控制但仍可进行碰撞检测

1.2 碰撞体和触发器

碰撞体和触发器是一体两面，一个物体在某一个时刻，只可能含有二者中的一个，要么是碰撞体要么是触发器，碰撞体和触发器主要用于物体之间的交互，碰撞体与触发器的区别在于物体是否具有物理特性，碰撞体是物理意义的碰撞，触发器是非物理意义的触碰。

1.2.1 碰撞体

1.2.1.1 碰撞体概念

  • 定义了物体的物理形状

  • 基本碰撞体

    • 包括：盒子碰撞体（Box Collider）、球体碰撞体（Sphere Collider）和胶囊碰撞体（Capsule Collider）
    • 最简单且节省资源
    • 不支持非等比缩放

  • 组合碰撞体(Compound Colliders)

    • 有多个碰撞体或含多个碰撞体的子物体
    • 常用于模拟外形复杂的物体

  • 注意

    • 碰撞体组件定义的是物体的物理形状，通常是看不见的，主要用于物理模拟和碰撞检测；网格过滤器组件定义的是物体的模型形状，通常是看得见的，主要用于视觉呈现和管线交互。如下图所示，虚线部分是物理形状，黑色部分是模型形状：

    • 基本碰撞体不支持非等比缩放

    • 碰撞体和触发器在一个组件中，勾选Is Trigger选项后碰撞体变为触发器

1.2.1.2 碰撞体分类
  • 分类依据:(1)是否具有刚体组件(2)刚体组件的动力学设置
  • 分为3类:(1)静态碰撞体(2)刚体碰撞体(3)动力学刚体碰撞体
1 静态碰撞体
  • 没有挂载刚体组件
  • 物理系统会假定其不会移动和改变位置
  • 不易随意修改状态
  • 常用于:关卡中的固定部分,如地形和障碍物
2 刚体碰撞体
  • 挂载了普通刚体组件
  • 物理系统会一直模拟计算其物理状态
  • 会对碰撞以及脚本施加的力做出反应
3 动力学刚体碰撞体
  • 挂载了刚体组件,勾选了动力学设置
  • 可在脚本中直接修改位置进行移动,不会对碰撞、力和速度的变化做出反应
  • 可随时改变动力学设置，需要移动时勾选，需要受物理影响时取消
  • 通常用于需要改变位置或状态的碰撞体上

1.2.2 触发器

  • 勾选碰撞体组件的Is Trigger属性,其变为触发器
  • 不再是物理上的固体,别的物体可以穿过或与其重叠
  • 另一物体进入触发器物体的范围时,脚本调用OnTriggerEnter()方法
  • 分类与碰撞体分类相似

1.2.3 碰撞和触发事件表

前面知道，一个物体，要么挂载碰撞体，要么挂载触发器，并且碰撞体和触发器根据刚体、动力学设置分为三类，不同类别之间的碰撞体、触发器之间是否会发生碰撞、触发事件，两张表如下：

总结：

  • 碰撞事件：
    • 碰撞体之间才会发生
    • 两个碰撞体之间至少一个是无动力学的刚体，
  • 触发事件：
    • 至少有一个触发器
    • 不是静态触发器之间，或者静态触发器与碰撞体之间

1.3 休眠

  • 休眠（Sleeping）状态：刚体的移动速度和旋转速度已经慢于实现定义的阈值，保持一定事件，物理引擎假定其暂时稳定，在下次受力前，物理引擎不再反复计算该物体的运动，直到它被重新唤醒
  • 这是一种性能优化的方案，节约了运算资源
  • 大多数情况下，刚体的休眠和唤醒都是自动进行的

1.4 物理材质

物理材质（Physics Materials）用于设置物体表面的摩擦和弹性，以此来模拟物体表面材料的特性，实现碰撞体之间发生交互时的物理效果。

  • 与碰撞体配合使用，属于碰撞体组件的Material选项，不指定物理材质时，碰撞体具有默认的物理材质

  • 创建物理材质：在Project窗口中单击鼠标右键，选择Create→Physics Material

  • 物理材质的参数：

    1. 动态摩擦系数（Dynamic Friction）
    2. 静态摩擦系统（Static Friction）
    3. 弹性系数（Bounciness）
    4. 摩擦力系数算法（Friction Combine）:两个物体表面都含摩擦系数时，计算摩擦系数的方法：（1）平均值（2）最大值（3）最小值（4）相乘
    5. 弹性系数算法（Bounce Combine）：参考4

1.5 层

  • 与标签分为多个类似，层也有多个
  • 每个物体属于某一个层
  • 通过对不同层之间的设置，达到物体间相互影响或不影响
  • 层碰撞矩阵（Layer Collision Matrix）
    • 位置：Edit→Project Settings→Physics
    • 列出来所有层之间是否会发生碰撞

1.6 物理关节

  • 关节（Joints）指一种物理上的链接关系，如门的合页、笔记本屏幕与键盘间的铰链等
  • Unity提供多种类型的关节，用于不同情景
  • 关节提供了许多选项和参数，以对应特定功能

1.7 射线检测

  • 射线检测指在游戏世界中发射一条虚拟的“射线”，并观察该射线是否击中了某个物体以及具体击中了哪个位置
  • 此“射线”并非我们印象中的射线，可以根据需求设置其发射位置、方向、长度、种类（直线状的、球形的、盒子形的等）

1.8 角色控制器

控制角色有两种方法：

  1. 脚本：直接用脚本改变物体的位置和朝向
  2. 刚体：通过刚体施加力和改变速度控制角色的行为

脚本控制：

  • 优点：灵活可控，各种基本功能通过脚本代码来实现
  • 缺点：复杂困难，代码编写不易

刚体控制：

  • 优点：简单易行，大部分功能可以借助物理系统实现
  • 缺点：不易考虑周全，物理环境复杂，多种力的作用不好分析

传统的动作游戏均采用脚本控制角色。

1.9 3D物理系统和2D物理系统

  • Unity含两套完全独立的物理系统：3D物理系统和2D物理系统
  • 二者截然不同，无太多联系
  • 虽无本质关系，两套物理系统的概念都能对的上

2 物理系统脚本编程

2.1 对刚体施加作用力

2.1.1 向重心施加力

  • 方法名：Rigidbody.AddForce()
  • 传参方式：Rigidbody.AddForce(Vector3 force, ForceMode mode（可选）)
  • 功能：对刚体施加作用力，施加位置时刚体的重心，力是一个向量，方向是向量的方向，力在每个坐标轴上的值对应向量每个坐标轴的值，力的单位是牛，持续时间为一个物理帧间隔（默认0.002秒）
  • 其中参数mode为力的模式，定义如下：

2.1.2 向指定位置施加力

  • 方法名：AddForceAtPosition()
  • 传参方式：void AddForceAtPosition(Vector3 force, Vector3 position, ForceMode mode)
  • 功能：向指定位置position施加作用力

2.2 刚体的速度

2.2.1 线性速度

  • 属性名：Rigidbody.velocity
  • 类型：Vector3
  • 含义：表示刚体的线性速度
  • 注意：新版Unity6，貌似该属性名更新为了linearVelocity

2.2.2 角速度

  • 属性名：Rigidbody.angularVelocity
  • 类型：Vector3
  • 含义：表示刚体的角速度，3维向量中的每个维度的值代表刚体绕世界坐标系x轴、y轴、z轴的旋转速度，单位为弧度/秒，即3.14代表半圈/每秒
  • 刚体具有角阻尼（Angular Drag），会使物体的角速度逐渐减缓至0

2.3 刚体的重心

由于游戏引擎中的物体并不受真实世界的限制，物体的重心并不是固定的，是可以调节的，可以通过脚本调节。

  • 属性名：Rigidbody.centerOfMass
  • 类型：Vector3
  • 含义：表示物体的重心
  • 对物体施加力时，施加力的位置（与物体重心的相对位置）不同，效果也不同；

2.4 射线的使用方法

  • 射线检测在游戏开发中应用广泛，比如：判断玩家是否接触墙壁，可以往墙壁方向发射射线
  • 射线检测基于物理系统，需要与碰撞体和触发器配合发挥作用
  • 射线有多种类型，其中最常用的是直线型射线

2.4.1 直线型射线

  • 直线型射线结构体

    • 类型及参数：Ray ray(Vector3 origin, Vector3 direction)
    • 含义：用于表示一个直线型射线，类型为Ray，含有两个Vector3类型的向量参数：起点(origin)和方向(direction)

  • 碰撞信息结构体：

    • 类型：RaycastHit
    • 参数：碰撞体的各类信息，如transform、collider、gameObject等
    • 含义：用于存储射线所碰撞的物体的各种信息

  • 发送直线型射线：

    • 方法名：Raycast()

    • 传参方式：

      1. bool Raycast(Vector3 origin, Vector3 direction,out RaycastHit hitInfo（可选）, float maxDistance（可选）, int layerMask（可选）);
      2. bool Raycast(Ray ray, out RaycastHit hitInfo（可选）, float maxDistance（可选）, int layerMask（可选）);

    • 功能：从指定起点向指定方向发射射线，返回射线是否碰撞到物体，若碰撞到物体，碰撞信息保存在结构体hitInfo中；起点和方向由origin和direction或ray所示，后三个可选参数分别为：碰撞信息、射线最大距离、碰撞遮罩（或者隐藏）层

除了直线型射线，还有球形射线、盒子射线、胶囊体射线，如下图所示：

发射这些射线的方法与发射直线型射线类似，只是个别参数发生了改变。

  • 发射球形射线：
    • 方法名：SphereCast()
    • 传参方式：bool SphereCast(Ray ray, float radius（球体半径）, out RaycastHit hitInfo（可选）)
  • 发射盒子射线：
    • 方法名：BoxCast()
    • 传参方式：bool BoxCast(Vector3 center（中心点）, Vector3 halfExtents（边长一半）, Vector3 direction, out RaycastHit hitInfo（可选）, Quaternion orientation（可选，盒子朝向）)
  • 发射胶囊提射线：
    • 方法名：CapsuleCast()
    • 传参方式：bool CapsuleCast(Vector3 point1（上中心点）, Vector3 point2（下中心点）, float radius（半径）, Vector3 direction, out RaycastHit hitInfo（可选）, float maxDistance（可选）)

2.4.3 穿越射线

又是需要射线遇到第一个物体时不停，继续向前，穿越多个物体，这类射线称之为穿越射线。

  • 方法名：RaycastAll()
  • 传参方式：
    • RaycastHit[] RaycastAll(Vector3 origin, Vector3 direction, float maxDistance)
    • RaycastHit[] RaycastAll(Ray ray, float maxDistance, int layerMask)
  • 功能：返回一个RaycastHit数组，存储直线型射线沿途碰到的所有碰撞信息
  • 其他类型的穿越射线方法名：SpherecastAll()、BoxcastAll()和CapsulecastAll()

2.4.4 区域覆盖型射线

检测一个空间范围，不能用一条射线，而是用一个区域。

  • 球形覆盖检测：

    • 方法名：OverlapSpere()

    • 传参方式：Collider[] OverlapSphere(Vector3 position, float radius, int layerMask)

    • 功能：返回原点为positon、半径为radius的球形区域内，满足一定条件的碰撞体集合

  • 盒子形覆盖检测：

    • 方法名：OverlapBox()
    • 传参方式：Collider[] OverlapBox(Vector3 center, Vector3 halfExtents, Quaternion orientation, int layerMask)

  • 胶囊体覆盖检测：

    • 方法名：OverlapCapsule()
    • 传参方式：Collider[] OverlapCapsule(Vector3 point0, Vector3 point1, float radius, int layerMask)

2.4.5 射线可视化

由于射线往往时看不见的，我们可以使用可视化方法将其画出来，但最终完成的游戏中，是不会看到这些辅助射线的。

在默认情况下，该辅助线仅在编辑器的场景窗口中可见。如果要在Game窗口中看到它，则需要单击Game窗口右上角的Gizmos（辅助线框）按钮。

  • 射线可视化
    • 方法名：DrawRay()
    • 传参方式：void DrawRay(Vector3 start, Vector3 dir, Color color, float duration(可选，持续时间，不写则持续一帧))
  • 线段可视化：
    • 方法名：DrawLine()
    • 传参方式：void DrawLine(Vector3 start, Vector3 end, Color color, float duration)

2.6 层和层遮罩

有时候，需要射线仅仅被某些物体阻挡，并且可以穿过其他物体，这时候，需要用到层（Layer）和层遮罩（LayerMask）的概念。

在脚本代码中，一个物体的层是int类型，层的名字可以是整数也可以是字符串。

  • 层：

    • 属性名：gameObject.layer

    • 类型：int类型

    • 功能：表示当前物体的层

通过类LayerMask的其方法、属性，来实现对层的操作。

  • 遮罩层：

    • 方法名：LayerMask.GetMask()

    • 传参方式：LayerMask.GetMask(层1，层2，层3...)

    • 功能：指定隐藏若干个层，返回一个int类型

  • 修改层：

    • 方法名：LayerMask.NameToLayer()

    • 传参方式：LayerMask.NameToLayer(层名)

    • 功能：修改物体所在的层

2.7 FixedUpdate()

对于Update()函数，我们已经十分熟悉，它是事件更新函数，一帧一帧地更新，并且更新地频率是不稳定的，帧之间的时间间隔Time.deltaTime不是固定的；FixedUpdate()函数可以大致理解为稳定的Update()函数，其帧与帧之间的时间间隔Time.fixedDeltaTime是固定的，默认为0.02秒，可以通过选择主菜单的Edit→Project Settings→Time来修改。

物理系统对于时间是敏感的，由于物理世界的时间是连续的，而游戏引擎中的时间是间断的、以一帧的时间间隔进行更新的，因此，总有一些需要检测的事件发生在两个帧之间，这影响了模拟物理世界的效果。要实现好的物理模拟，需要：（1）更新频率高（相应的，硬件开销也大了）（2）更新频率稳定。

当我们处理一些受物理系统控制的运动物体的更新时，相比于使用Update()函数更新，使用FixedUpdate()函数更好。
